# eur-oods-pages

本项目用于归档和发布多个独立的 HTML 页面。所收录的页面均为静态网页，不依赖特定后端服务，可以直接在浏览器中打开查看。

## 项目简介

`eur-oods-pages` 是一个以页面归档为主要目的的仓库。这里存放的 HTML 文件通常用于展示、测试或记录特定时间点的页面状态。每个页面彼此独立，互不依赖，适合作为静态资源集合进行管理与分发。

## 目录说明

```
eur-oods-pages/
├── index.html          # 项目根页面，用于导航或索引
├── pages/              # 存放主要归档页面的目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/             # 公共资源目录（CSS、JS、图片等，视需要而定）
```

- `index.html`：可选的入口页面，提供到各归档页面的链接。
- `pages/`：核心目录，所有独立归档的 HTML 页面均放置于此。
- `assets/`：如果页面共享样式、脚本或图片资源，可统一放在此目录下。

## 页面归档说明

- 每个 HTML 页面均为独立完整的静态文件，包含其所需的样式与结构。
- 归档页面的内容可能来源于历史快照、测试用例或临时展示需求，不代表长期维护的网站。
- 页面命名应尽量简洁明了，便于识别内容主题或用途。
- 归档后一般不进行频繁修改，如需更新可新增版本或替换文件。

## 维护说明

- 本仓库不针对任何具体域名或网站，仅作为页面集合的存储与发布场所。
- 欢迎提交 Pull Request 添加新的归档页面，但请确保页面内容合法合规且不包含敏感信息。
- 如需删除或替换已有页面，建议先通过 Issue 进行讨论。
- 仓库维护者保留对页面收录的最终决定权。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/eur-oods-pages.git
   ```
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意 HTML 文件即可查看。
3. 也可通过 GitHub Pages 等服务进行在线预览（需自行配置）。
